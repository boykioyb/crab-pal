# v0.0.6 — OAuth refresh actually refreshes 🔑

Follow-up to v0.0.5. The `refreshAuthEnvVars()` path did its job, but a separate bug upstream of it meant some users never hit the refresh code at all.

## Fixes

### 🔐 Claude OAuth tokens no longer get stuck un-refreshable
If Anthropic's token endpoint omitted `expires_in`, CrabPal stored the credential with no expiry. `isTokenExpired()` then returned `false` forever, so auto-refresh never fired — the subprocess kept spawning with a stale token and the API returned 401 regardless. Now:
- `refreshClaudeToken` and `exchangeClaudeCode` fall back to a 1h expiry when `expires_in` is missing, so the next refresh cycle kicks in well before the server-side expiry.
- `getValidClaudeOAuthToken` treats a credential with missing `expiresAt` as "needs refresh", self-healing anyone already stuck in that state.

### 🪵 Settings → Logs works in packaged builds
File logging is now enabled by default in release builds (info-level), so the in-app log viewer has content to show without launching from a terminal. Console output still stays off unless `--debug` is passed.
